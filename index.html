闭包

闭包的概念，

宏观：js中所有的函数都能形成一个闭包
微观：函数里包裹其他的函数，并返回这个函数，就形成闭包（这也是开发中经常用到的思想）
 另一种说法，函数体以及函数所处的环境构成的总合体称为闭包

闭包的初级：

　function f1(){
　　　　n=999;
　　　　function f2(){
　　　　　　alert(n);
　　　　}
　　　　return f2;
　　}
　　var result=f1();
　　result(); // 999

闭包的组成：

1函数体  （函数自身代码）
2环境  （函数的作用域）

闭包的作用：

1.对函数内部的变量起到了保护的作用
2.除了返回的函数以外，没有任何手段能够获取或者修改这个变量的值。

闭包的缺点

​ 就是常驻内存，会增大内存使用量，使用不当很容易造成内存泄露。

​ 要理解闭包。首先必须理解js的特殊的变量作用域，变量的作用域分为全局变量和局部变量

作用域：

​ 一个变量在定义的时候就有了一个作用区域；
​ 不在任何函数内声明的变量（函数内省略var的也算全局）称作全局变量
​ 在函数内声明的变量具有函数作用域，属于局部变量

作用域链：

​ 多个函数嵌套，子域可以访问父域的变量，形成的链式结构；（如图）；

对于分析作用域的面试题：

​ 只需要看函数是在哪个环境中创建的，而不管函数是在哪个环境中调用的

面试题

var name = "The Window";   
　　var object = {   
　　　　name : "My Object",   
　　　　getNameFunc : function(){   
　　　　　　return function(){   
　　　　　　　　return this.name;   
　　　　　};   
　　　　}   
};   
alert(object.getNameFunc()());  //The Window

闭包的运用

1.匿名函数自调用

2.实现封装/模块化代码

​      var person= function(){ 
   //变量作用域为函数内部，外部无法访问 
​         var name = "default"; 
​         return { 
​            getName : function(){ 
​                return name; 
​         }, 
​         setName : function(newName){ 
​              name = newName; 
​         }

​    console.log(person.name);//直接访问，结果为undefined 
​    console.log(person.getName()); //default

闭包内存问题的解释：

 function demo（）{
   var arr = [];
   for(var i = 0; i < 10; i++){

​    arr[i] = function(){ return i; }   

  }

​     return arr;

}
var arr = demo();
arr(3)();

思路：通过for循环把需要返回的函数存到了一个数组中，和存到内存中一个性质；
